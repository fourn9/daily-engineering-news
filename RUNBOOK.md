# RUNBOOK — エンジニアリング・モーニングブリーフ生成ルーティン用

このファイルは、毎朝の「エンジニアリング・モーニングブリーフ」を生成するクラウドルーティンが
**ブリーフを書き始める前に必ず読む**運用ガイドです。ルーティン本体（claude.ai 側）の
プロンプトから「生成前に必ず本リポジトリの `RUNBOOK.md` を読み、その重複除外ルールに従う」
と参照させてください。ルールを変えたいときは、この 1 ファイルを編集して commit するだけで済みます。

## 生成フロー（概要）

1. リポジトリはチェックアウト済み。まず**過去分の照合**（下記「重複除外ルール」1)を行う。
2. 本日（JST）のエンジニアリング動向を収集し、下記カテゴリに整理してブリーフHTMLを生成する。
3. `YYYY-MM-DD/index.html` を追加し、トップ `index.html` の
   `<!-- INSERT-NEXT-BRIEF-HERE -->` 直下に新しい日付の `<li>` を先頭追記する。
4. commit & push。反映URL: https://fourn9.github.io/daily-engineering-news/<DATE>/

カテゴリ（既存踏襲）: 🎯 今日のハイライト ／ 🤖 AI / LLM プラットフォーム ／
📐 アーキテクチャ / 設計 ／ ☁️ インフラ / クラウド / 開発ツール ／
👥 エンジニアリング組織 / プラクティス ／ 🌟 GitHub Trending / 注目OSS ／
🇯🇵 日本語コミュニティ動向。

## 重複除外ルール（最重要 / 今回の是正点）

**目的**: 過去に公開したブリーフの項目を、本日分として再び取り上げない。
「その日でないニュース」を新規のように出すのを止める。

1. **過去分を必ず先に読む**
   書き始める前に、直近の公開済みブリーフ（最低でも直近 3 日、可能なら 7 日分）の
   `YYYY-MM-DD/index.html` を読み、**既出の見出し（`<h3>`）と参照元URL（`<a href>`）**を
   一覧化しておく。これと照合してから本日の項目を選ぶ。

2. **本日性で絞る**
   掲載してよいのは、**発表・公開・更新が本日（JST）付**の項目だけ。
   前日以前に既出のリリース／記事の焼き直しや、過去日付のまとめ（例:「週刊AI駆動開発 ○月○日」の
   ように本日より前の日付が入った定期ダイジェスト）は**載せない**。
   GitHub Trending / 注目OSS も、既に過去号で取り上げたリポジトリは再掲しない。

3. **進行中トピックは「新展開があるときだけ」**
   すでに扱った継続トピック（同じプロダクト・同じ論文シリーズ等）は、**本日新たな進展が
   ある場合のみ**掲載する。その場合は必ず `続報` タグを付け、リード文を
   「**何が新しいか**」から書き出す。前日と同じ内容の言い換えは載せない。

4. **同一項目の再掲・二重掲載の禁止（号内の日英・カテゴリ跨ぎ含む）**
   同一 URL、または実質的に同じ見出し（表記ゆれ・**和英差**を正規化して照合）の項目は
   再掲しない。**同じ号（同じ日）の中でも、同じ出来事を英語見出しと日本語見出しで二重に、
   あるいは複数カテゴリに二重掲載しない**（例: 買収・新機能発表を英語ソースと日本語ソースで
   別項目にしない）。代表 1 件に集約し、必要なら本文内で両ソースに触れる。

5. **迷ったら載せない**
   本日性が確認できない、既出かどうか判断がつかない項目は除外する。

### 照合の実務メモ

- 過去 `index.html` からの抽出例:
  `grep -oE '<h3[^>]*>.*</h3>' <DATE>/index.html`（見出し）、
  `grep -oE 'href="https?://[^"]+"' <DATE>/index.html`（URL）。
- URL は末尾のトラッキングパラメータ（`?utm_...` 等）を落として比較する。
- 見出しは記号・空白・**和英**を無視した実質一致で判定する（同じ出来事の EN/JA は同一とみなす）。
